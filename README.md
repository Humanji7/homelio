# Homelio Test Assignment

Локальное решение для детерминированного автозаполнения трех тендерных DOCX-документов по тестовым JSON-данным.

## Что сдается

Репозиторий содержит:

- CLI `homelio` для генерации документов;
- явный маппинг полей в `config/mappings/documents.yaml`;
- заметки по reverse engineering в `notes/`;
- архитектурную записку в `notes/architecture.md`;
- итоговые DOCX в `output/`.

## Требования

- Python 3.12+
- `uv`

## Локальный запуск

1. Установить зависимости:

```bash
uv sync --extra dev
```

2. При необходимости посмотреть плейсхолдеры в шаблонах:

```bash
uv run homelio inspect-placeholders "materials/Материалы/02. Шаблоны"
```

3. Сгенерировать итоговые документы:

```bash
uv run homelio generate \
  --document-date 2026-03-26 \
  --outgoing-number "№2603/1"
```

Команда по умолчанию:

- берет входные JSON из `materials/Материалы/04. Пример данных JSON`;
- использует шаблоны из `materials/Материалы/02. Шаблоны`;
- читает правила заполнения из `config/mappings/documents.yaml`;
- записывает результаты в `output/`.

Если `--outgoing-number` не передан, поле исходящего номера остается пустым. Генератор не подставляет значение по догадке, потому что его нет во входных JSON.

## Ожидаемый результат

После успешного запуска в `output/` должны лежать 3 файла:

- `01_Анкета_участника_заполнено.docx`
- `02_Заявка_на_участие_в_закупке_заполнено.docx`
- `03_Предложение_о_цене_договора_заполнено.docx`

## Проверка решения

```bash
uv run pytest
uv run ruff check
uv run homelio generate --document-date 2026-03-26 --outgoing-number "№2603/1"
```

## Структура репозитория

- `src/homelio` — CLI и генератор DOCX.
- `config/mappings` — машиночитаемый маппинг плейсхолдеров и табличных блоков.
- `materials/Материалы` — шаблоны DOCX, эталонные примеры и тестовые JSON.
- `notes` — reverse engineering, подготовительные заметки и архитектурная записка.
- `output` — артефакты финальной генерации.

## Ограничения текущей версии

- Поддерживаются ровно 3 документа из тестового задания.
- Поля заполняются только из явных источников `profile`, `tender`, `calc`, `system`.
- Решение не извлекает данные из внешних систем и не использует LLM для подстановки значений.
