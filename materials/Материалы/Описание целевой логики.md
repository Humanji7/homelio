# Краткое описание целевой логики

Мы не ожидаем, что документы будут заполняться «магически» только за счёт LLM. Наоборот, для типовых полей предпочтительна детерминированная подстановка по правилам и маппингу.

Целевой контур:
1. Есть 3 источника данных: `profile`, `tender`, `calc` (+ системные поля вроде текущей даты).
2. Для каждого документа есть набор правил, какое поле из какого источника брать.
3. На выходе получается заполненный DOCX максимально близкий к эталону.
4. Если данных нет, система не должна выдумывать их.
5. Расширение на новые документы должно происходить через добавление шаблона + маппинга, а не через переписывание всей программы.

Использование AI допустимо для извлечения полей из входящих документов, но это необязательная часть и скорее bonus.
