# BRIEF

## Problem

Нужно собрать локальное и надежное решение, которое генерирует 3 DOCX-документа по входным JSON-данным без выдумывания значений и без ручной правки после генерации.
Дополнительно нужно подготовить проект к сдаче: оформить публичный репозиторий и собрать чистый артефакт для отправки.

## Desired Behavior

- Система принимает данные из источников `profile`, `tender`, `calc`.
- Для каждого документа правила заполнения отделены от кода и описаны явно.
- При отсутствии данных генератор оставляет пустое значение или понятный маркер, а не подставляет догадки.
- Итоговые документы визуально и структурно максимально близки к эталонам.
- Проект опубликован как публичный GitHub-репозиторий.
- Есть чистый артефакт сдачи, который можно приложить или отправить ссылкой.

## Non-goals

- Не делаем интеграции с реальным сайтом, CRM, API или приватными сервисами.
- Не строим "магическое" решение только на LLM там, где хватает детерминированного маппинга.
- Не расширяем решение за рамки текущих 3 документов до появления рабочего базового контура.

## Affected Areas

- `materials/Материалы` — шаблоны, примеры и тестовые JSON.
- `src/homelio` — CLI, анализаторы шаблонов, генератор документов.
- `config/mappings` — правила и источники полей.
- `notes` — архитектурная записка и промежуточные наблюдения.
- `output` — результаты генерации для сдачи.
- Корень репозитория — git-метаданные и упаковочные артефакты сдачи.

## Constraints

- Решение должно запускаться локально по инструкции из `README.md`.
- Нельзя выдумывать данные, использовать реальные реквизиты или компенсировать логику ручной правкой DOCX.
- Подготовительный этап должен остаться узким: только среда, каркас и инструменты анализа.
- Для публикации не включать виртуальное окружение, кеши и временные локальные файлы.
